Introduction & Learning Outcomes

Introduction

The Android UI is constructed as a hierarchy of Views (widgets like buttons) and ViewGroups (containers/layouts). Your UI is typically defined in XML files within the res/layout directory, which is then inflated (loaded into memory) and manipulated by your Activity or Fragment class written in Java.

Learning Outcomes

  • Understand Material Design principles for high-quality Android apps.
  • Master the use of key Layouts (ViewGroups) like ConstraintLayout and LinearLayout.
  • Implement and manage common UI Controls (Views) and handle user interaction using Java.
  • Create adaptive designs using device-independent units.

Lesson 1. Fundamental Android Design

Material Design Principles

  1. Tangible Surfaces: UI elements should look and behave like real-world material, using elevation (shadows) to indicate hierarchy and depth.
  2. Intentional Graphics: Use bold color, typography, and clean spacing to direct the user's focus.
  3. Usability: Design for the user's hand and thumb reach. Ensure all interactive elements have a minimum target size of 48 dp.

Units of Measurement

  • dp (Density-Independent Pixels): Used for all dimensions (width, height, margin, padding). It scales to ensure UI elements appear the same physical size across different screen densities.
  • sp (Scale-Independent Pixels): Used only for text sizes. It scales based on the user's system font size preference for accessibility.
  • Avoid px (Pixels): This unit does not scale and will result in inconsistent layouts.

Lesson 2. UI Layouts

Layouts are ViewGroups defined in XML that dictate how child Views are positioned.

Common Layout Attributes

AttributeDescriptionCommon Values
android:idUnique ID for referencing in Java code.@+id/my_element_id
android:layout_widthHorizontal size.wrap_content, match_parent
android:layout_heightVertical size.wrap_content, match_parent
android:layout_marginExternal spacing outside the View's boundary.e.g., 16dp
android:paddingInternal spacing between the View's boundary and its content.e.g., 8dp

Main Layout Types

1. LinearLayout

  • Arranges children sequentially in a single row or column.
  • XML Key: android:orientation="horizontal" or "vertical".
  • Weight: android:layout_weight allows you to distribute space proportionally.

2. ConstraintLayout

  • The recommended, flexible layout for non-nested UIs.
  • Views are positioned by defining constraints relative to sibling views, the parent, or virtual helpers.
  • Key Benefit: Flat view hierarchy leads to better performance.

3. FrameLayout

  • Views are stacked on top of each other. Often used to hold a single Fragment.

4. ScrollView & RecyclerView

  • ScrollView: Used for vertical scrolling of small amounts of content. Can only contain one direct child.
  • RecyclerView: Used for efficiently displaying large, dynamic lists of data (view recycling).

Assessment Task

Task: Create a simple login screen Activity.

Requirements:

  1. Use a ConstraintLayout as the root container.
  2. Include two EditText fields for email and password.
  3. Include a Switch control for "Keep me logged in."
  4. Include a Button labeled "Login."
  5. Java Implementation: In your MainActivity.java, implement the button's OnClickListener. When the button is clicked, retrieve the text from the email EditText and display it in a Toast message.
// Partial Java Code for Task (in MainActivity.java)
public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main); // Load the XML layout

        // 1. Link Controls to Java objects
        final EditText emailEditText = findViewById(R.id.et_email);
        Button loginButton = findViewById(R.id.btn_login);

        // 2. Implement the button click listener
        loginButton.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                // Retrieve the value from the EditText
                String email = emailEditText.getText().toString();

                // Display the retrieved value
                Toast.makeText(MainActivity.this,
                               "Attempting login for: " + email,
                               Toast.LENGTH_LONG).show();
            }
        });
    }
}

Summary

  • Android UI is a hierarchy of Views (controls) and ViewGroups (layouts).
  • Material Design is the guiding principle for creating visually consistent and intuitive apps.
  • Layouts like ConstraintLayout are essential for building flexible and efficient UIs.
  • UI Controls (TextView, EditText, Button, CheckBox, etc.) allow user interaction and display content.
  • All UI dimensions are preferably defined using dp (for size/layout) and sp (for text) to ensure proper scaling across various devices.
