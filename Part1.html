Part 1: The User Interface (UI) - XML Layout

The user interface is defined in an **XML** file, which serves as a blueprint for the screen's layout. This file describes what elements appear on the screen and how they are arranged. It’s important to understand that XML is a markup language, not a programming language, so it's all about structure and visual attributes.

The Code

Below is the code for the `activity_main.xml` file. It's built around a `` which acts as the main container. This layout arranges all the views vertically, making it simple to place a title, two input fields, and a button. 

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    android:gravity="center"
    android:padding="16dp"
    tools:context=".MainActivity">

    <!-- Title of the activity -->
    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="User Login"
        android:textSize="24sp"
        android:textStyle="bold"
        android:layout_marginBottom="32dp" />

    <!-- Input field for the username -->
    <EditText
        android:id="@+id/editTextUsername"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="Username"
        android:inputType="text"
        android:padding="12dp"
        android:layout_marginBottom="16dp"
        android:background="@drawable/rounded_edittext" />

    <!-- Input field for the password -->
    <EditText
        android:id="@+id/editTextPassword"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:hint="Password"
        android:inputType="textPassword"
        android:padding="12dp"
        android:layout_marginBottom="24dp"
        android:background="@drawable/rounded_edittext" />

    <!-- Login button -->
    <Button
        android:id="@+id/buttonLogin"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        android:text="Login"
        android:padding="12dp"
        android:backgroundTint="#007BFF"
        android:textColor="#FFFFFF" />

</LinearLayout>

Key Attributes Explained

Each element has a set of attributes that control its appearance and behavior. Here are some of the most important ones:

  • `android:id`: A unique identifier for each view. This is how the Java code finds and interacts with the UI elements.
  • `android:layout_width` & `android:layout_height`: These define the size of the view. `match_parent` means the view will expand to fill the size of its parent container.
  • `android:text`: Sets the text for a `TextView` or `Button`.
  • `android:hint`: Provides a placeholder text inside an empty input field, like "Username" or "Password."
  • `android:inputType`: Specifies the type of data the user should enter. Using `textPassword` for the password field automatically hides the characters as the user types.
