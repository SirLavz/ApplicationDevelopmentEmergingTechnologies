Introduction to Android and Android Studio

Learning Objectives

Upon completion of this lesson, you will be able to:

  • Explain the role and significance of **Android** as a mobile operating system.
  • Describe the key components and architecture of the Android ecosystem.
  • Successfully set up the **Android Studio** Integrated Development Environment (IDE) on your computer.
  • Differentiate between an **Android Virtual Device (AVD)** and a physical device for app testing.
  • Create a basic "Hello, World!" Android project from scratch using Android Studio.
  • Navigate the Android Studio interface and identify its main windows and tools.

Lesson Content

1. What is Android?

Android is a Linux-based, open-source mobile operating system primarily designed for touchscreen mobile devices such as smartphones and tablets. Developed by Google and the Open Handset Alliance, it is the most widely used mobile operating system in the world. Its open-source nature allows developers to create and distribute applications for a massive global audience.

  • **Open-Source**: The source code is publicly available, allowing for customization and a vibrant developer community.
  • **Versatility**: Used on a wide range of devices, including phones, tablets, smart TVs, and smartwatches.
  • **App Ecosystem**: The Google Play Store serves as the primary distribution channel for millions of applications.

2. What is Android Studio?

Android Studio is the official Integrated Development Environment (IDE) for Android application development. It is the go-to tool for developers building apps for the Android platform. Based on IntelliJ IDEA, Android Studio provides a robust set of features specifically tailored for Android.

  • **Code Editor**: A powerful editor with code completion, refactoring, and linting to catch errors.
  • **Layout Editor**: A visual designer for building user interfaces (UIs) by dragging and dropping UI components.
  • **Emulator**: The Android Virtual Device (AVD) manager allows you to run and test your app on virtual devices.
  • **Debugging Tools**: Features like the debugger, profilers, and logcat to monitor and troubleshoot your application.
  • **Build System**: The Gradle build system automates the process of compiling, packaging, and signing your app.

3. Setting Up Your Development Environment

To begin Android development, you must first install and configure Android Studio.

Step-by-step installation process:

  1. Download Android Studio: Go to the official Android Developer website and download the latest version of Android Studio for your operating system (Windows, macOS, or Linux).

  2. Installation: Follow the on-screen instructions of the installer. This will install the core IDE and the Android SDK (Software Development Kit).

  3. Initial Setup Wizard: Upon the first launch, Android Studio will guide you through a setup wizard to download essential components, including the latest SDK, system images for the emulator, and build tools. Select the default options unless you have a specific need to change them.

  4. Creating a Virtual Device (AVD):

    • From the Android Studio welcome screen, select "More Actions" -> "Virtual Device Manager".
    • Click "Create Virtual Device" and choose a hardware profile (e.g., Pixel 7).
    • Select a system image (e.g., Android 13/Tiramisu). This is the version of Android that will run on your virtual device. Download it if you haven't already.
    • Give your AVD a name and click "Finish". Your emulator is now ready for use.

4. Your First Android Project: "Hello, World!"

This is the standard starting point for any new programmer. It demonstrates the basic steps of creating, running, and viewing a simple app.

Step-by-step project creation:

  1. From the Android Studio welcome screen, select **"New Project"**.

  2. In the "New Project" dialog, choose the **"Empty Views Activity"** template.

  3. Click **"Next"**.

  4. Configure your project:

    • **Name**: MyFirstApp
    • **Package name**: com.example.myfirstapp (this must be unique)
    • **Save location**: Choose a folder on your computer.
    • **Language**: Java
    • **Minimum SDK**: Choose an API level (e.g., API 24: Android 7.0). This determines the oldest Android version your app can run on.

  5. Click **"Finish"**. Android Studio will now build your project. This may take a few minutes as it downloads dependencies.

5. Navigating the Android Studio Interface

Once the project is built, you will see the main Android Studio interface. Key windows to familiarize yourself with:

  • **Project Window** (left pane): Shows the file structure of your project. The most important folders are:
    • app/java: Contains your Java source code files.
    • app/res/layout: Contains your XML layout files for the UI.
    • app/res/drawable: Contains image assets.
  • **Editor Window** (center): Where you write and edit your code (both Java and XML).
  • **Design/Split/Code View** (top-right of Editor): For XML layout files, you can switch between a visual Design view, a text-based Code view, or a Split view showing both.
  • **Logcat Window** (bottom): Displays system messages and logs from your app, which is crucial for debugging.
  • **Emulator/Device Selector** (top toolbar): A dropdown to choose which device (AVD or physical) to run your app on.
  • **Run Button** (green play icon, top toolbar): Builds and launches your app on the selected device.

Exercise:

  1. After creating your project, click the green "Run" button in the toolbar.
  2. If prompted, choose the AVD you created earlier.
  3. The emulator will launch and your app will be installed. You should see a simple screen with "Hello, World!" displayed, confirming your setup is correct.
