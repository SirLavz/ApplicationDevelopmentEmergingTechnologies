SQLite Database: CRUD Operations in Android (Java)

Prerequisite:

This tutorial uses the **`SQLiteOpenHelper`** class and **Java** to implement local database storage in Android Studio.

Phase 1: Database Setup and Creation (The Helper Class)

All core database logic is contained in a custom class that extends `SQLiteOpenHelper`. This class handles the actual creation of the database file and its tables.

Step 1: Create the `DatabaseHelper.java` Class

Create a new Java class (e.g., `DatabaseHelper`) and make it extend `SQLiteOpenHelper`. Define constants for the database name, version, table name, and columns.

import android.content.Context;
import android.database.sqlite.SQLiteDatabase;
import android.database.sqlite.SQLiteOpenHelper;

public class DatabaseHelper extends SQLiteOpenHelper {

    // Database constants
    public static final String DATABASE_NAME = "StudentDB.db";
    public static final int DATABASE_VERSION = 1;
    public static final String TABLE_NAME = "student_table";
    public static final String COL_1 = "ID";
    public static final String COL_2 = "NAME";
    public static final String COL_3 = "MARKS";

    // SQL statement to create the table
    private static final String CREATE_TABLE = "CREATE TABLE " + TABLE_NAME + " (" +
        COL_1 + " INTEGER PRIMARY KEY AUTOINCREMENT," +
        COL_2 + " TEXT," +
        COL_3 + " INTEGER" +
        ")";

    public DatabaseHelper(Context context) {
        super(context, DATABASE_NAME, null, DATABASE_VERSION);
    }

    // This method runs ONLY when the database file is first created
    @Override
    public void onCreate(SQLiteDatabase db) {
        db.execSQL(CREATE_TABLE);
    }

    // This method runs when DATABASE_VERSION is incremented
    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        db.execSQL("DROP TABLE IF EXISTS " + TABLE_NAME);
        onCreate(db);
    }
}

Phase 2: Implementing CRUD Operations (C - Create/Insert)

Step 2: Create the `insertData()` Method

This method uses the `ContentValues` class to package data and the `insert()` method of `SQLiteDatabase` to add a new row.

// Add this method to the DatabaseHelper class

public boolean insertData(String name, String marks) {
    SQLiteDatabase db = this.getWritableDatabase(); // Get write access
    ContentValues contentValues = new ContentValues();
    
    // Put data into the map (Key = Column Name, Value = Data)
    contentValues.put(COL_2, name);
    contentValues.put(COL_3, marks);

    // Insert the row. Returns -1 if it failed, or the ID of the row if successful
    long result = db.insert(TABLE_NAME, null, contentValues);
    
    return result != -1;
}

Phase 3: Implementing CRUD Operations (R - Read/Retrieve)

Step 3: Create the `getAllData()` Method

The `query()` or `rawQuery()` method returns a **`Cursor`** object, which points to the result set. We must iterate through the Cursor to retrieve data.

// Add this method to the DatabaseHelper class

import android.database.Cursor;

public Cursor getAllData() {
    SQLiteDatabase db = this.getWritableDatabase();
    // rawQuery allows you to run a direct SQL SELECT statement
    Cursor res = db.rawQuery("SELECT * FROM " + TABLE_NAME, null);
    return res;
}

Retrieving the Data (In your Activity):

DatabaseHelper myDb = new DatabaseHelper(this); // Instance
Cursor res = myDb.getAllData();

if (res.getCount() == 0) {
    // Handle case where no data is returned
    Toast.makeText(this, "No Records Found", Toast.LENGTH_SHORT).show();
    return;
}

// Iterate through the results
StringBuffer buffer = new StringBuffer();
while (res.moveToNext()) {
    // Column Index: 0=ID, 1=NAME, 2=MARKS (as defined in CREATE_TABLE)
    buffer.append("ID: " + res.getString(0) + "\n");
    buffer.append("Name: " + res.getString(1) + "\n");
    buffer.append("Marks: " + res.getString(2) + "\n\n");
}
res.close(); // IMPORTANT: Close the cursor to free resources
// Display buffer.toString() in an AlertDialog or TextView

Phase 4: Implementing CRUD Operations (U & D - Update/Delete)

Step 4: Create `updateData()` and `deleteData()`

These methods use the `update()` and `delete()` helper functions, which require a `WHERE` clause and a value array (`whereArgs`) to specify which row(s) to affect.

// Add this method to the DatabaseHelper class

public boolean updateData(String id, String name, String marks) {
    SQLiteDatabase db = this.getWritableDatabase();
    ContentValues contentValues = new ContentValues();
    contentValues.put(COL_1, id);
    contentValues.put(COL_2, name);
    contentValues.put(COL_3, marks);
    
    // Update data where ID matches the input ID
    db.update(TABLE_NAME, contentValues, "ID = ?", new String[] { id });
    return true; // Simple confirmation
}

public Integer deleteData(String id) {
    SQLiteDatabase db = this.getWritableDatabase();
    
    // Delete data where ID matches the input ID
    // Returns the number of rows deleted
    return db.delete(TABLE_NAME, "ID = ?", new String[] { id });
}

Summary and Best Practice

  • Always subclass **`SQLiteOpenHelper`** to manage database lifecycle.
  • Use **`ContentValues`** for Insert and Update operations for safety and ease.
  • Use a **`Cursor`** to retrieve results from any Read operation.
  • **Always close the `Cursor`** after you are finished reading data (`res.close();`).
  • When performing database operations in the Activity, ensure you handle exceptions and check return values (e.g., `result != -1` for insert, `res.getCount() == 0` for read).
